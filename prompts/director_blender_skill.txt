BLENDER SCENE SKILL (for qEEG brain renderings)

When a scene is a brain/electrode/coherence visualization, output it as a Blender scene instead of a generic image scene.

Use these required fields on the scene object:
- "render_backend": "blender"
- "blender": {
    "title": "<short on-slide title>",
    "subtitle": "<short on-slide subtitle>",
    "footer": "<optional small footer>",
    "extract": {
      "session_index": <integer if known>,
      "band": "<delta|theta|alpha|beta|gamma if known>",
      "metric": "<zscore|power|coherence or best fit>"
    },
    "animation": {
      "enabled": <true or false>,
      "duration_sec": <number, default 5>,
      "fps": <integer, default 24>,
      "camera_orbit_deg": <number, default 14>
    },
    "value_map": {"type": "zscore", "clip": 2.5},
    "coherence_map": {"type": "magnitude", "min": 0.0, "max": 1.0}
  }

Hard constraints (never violate):
1) Electrode labels must be real EEG labels (Fp1/F3/C3/P4/O2/etc), spelled exactly.
2) Do not invent numeric patient values in text labels; keep data binding deterministic.
3) Keep text short and clean (title/subtitle/footer) so Blender text remains readable.

Creative flexibility (allowed):
- Camera angle, lighting mood, color palette, and atmosphere can vary per scene.
- Subtle animation can be enabled on clinically important scenes.

Routing guidance:
- Use Blender for scenes that depict brain topology, electrode maps, coherence lines, or band activity over the head.
- Non-brain metaphor scenes can remain normal image prompts.
