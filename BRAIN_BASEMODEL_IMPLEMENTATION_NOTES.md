# BRAIN_BASEMODEL_IMPLEMENTATION_NOTES

## Scope Delivered
Implemented a canonical deterministic qEEG brain base model for Blender scenes, with style presets, idempotent scene updates, fixture-based regression rendering, and director/skill contract alignment.

**v2 update**: Replaced procedural UV sphere brain with real MRI-derived pial cortical surface mesh (brainder.org, CC-BY-SA 3.0). Electrodes now sit on the anatomical brain surface via BVH raycasting instead of floating on a detached perfect sphere.

## Architecture Decisions
- Canonical template asset remains: `blender_pipeline/assets/qeeg_template.blend`.
- Template is generated reproducibly from code: `blender_pipeline/scripts/build_template.py`.
- Per-scene mutation and rendering entrypoint is deterministic batch contract: `blender_pipeline/scripts/render_batch.py`.
- Runtime scene contract is built in Python before Blender execution: `core/blender_gen.py`.
- Validation harness and fixtures are generated by: `scripts/validate_brain_basemodel.py`.

## Brain Mesh Source and Processing (v2)
- **Source**: brainder.org MRI-derived pial surfaces (MNI152 template, CC-BY-SA 3.0)
- **Files**: `blender_pipeline/assets/brain_mesh/lh.pial.obj` (left hemisphere, 144K verts) and `rh.pial.obj` (right hemisphere, 145K verts)
- **Import pipeline** (`_import_brain_mesh()` in `build_template.py`):
  1. Import both OBJ files (version-safe: `wm.obj_import` then `import_scene.obj` fallback)
  2. Join into single mesh
  3. Reconsistent normals (`normals_make_consistent(inside=False)`)
  4. Recenter bounding box to origin
  5. Scale from MNI mm to Blender units (factor 0.0223, maps ~139mm width to ~3.1 BU)
  6. Decimate modifier (ratio 0.35, ~580K faces -> ~200K)
  7. Smooth modifier (8 iterations, factor 0.5) for medical-illustration polish
  8. Subdivision Surface modifier (viewport=0, render=1)
  9. Shade smooth

## Electrode Placement (v2)
- Electrodes placed via BVH raycasting onto the evaluated (modifier-applied) brain mesh
- `_brain_surface_point()` casts a ray from outside (10 BU along direction) toward origin
- Three-level fallback: `ray_cast()` -> `closest_point_on_mesh()` -> sphere at fallback radius
- Electrode sphere offset: +0.06 along surface normal
- Label text offset: +0.22 along surface normal
- This replaces the old fixed-radius sphere placement (r=2.22)

## Style Presets (Pure Visual Transforms)
Implemented in `render_batch.py` under deterministic style fields:
- `lighting_preset`: `clinical_glow`, `calm_precision`, `focus_contrast`
- `camera_preset`: `three_quarter_left`, `three_quarter_right`, `frontal`, `top_center`
- `palette`: `teal-amber`, `ice-white`, `cyan-orange`

Style changes are restricted to camera/light/fog/material mood and do not alter electrode identity, anchor coordinates, or numeric mappings.

## Deterministic Update Contract
- Scene contract version: `brain_basemodel_v1`.
- Contract emitted by `_build_scene_spec(...)` in `core/blender_gen.py` includes:
  - `extract` hints (`session_index`, `band`, `metric`, optional explicit paths)
  - `style`
  - deterministic maps for electrodes and coherence
- Render path is idempotent:
  - upsert materials by stable names
  - clear/rebuild coherence line objects by stable prefixes
  - no additive node drift
- Checkpoint hooks:
  - optional `--checkpoint-dir` renders after `style/text` and after `data-bound` phases.

## Apple Silicon / Performance Decisions
- GPU is now default on macOS unless explicitly disabled (`BLENDER_GPU=0`) in `core/blender_gen.py`.
- Blender scripts now select compute device using runtime enum introspection (not hard-coded assumptions) and prioritize Metal when available.
- Boolean solver selection also uses runtime enum introspection to stay API-compatible across Blender versions.
- Validation harness defaults to GPU mode and lower samples for fast deterministic regression.
- Runtime probe confirms Blender 5.0.1 sees `METAL` GPU device on this machine and uses it; no separate ANE render-device enum is exposed in Cycles device settings.

## Safe Extension Guidance
- To add a new visual style, only add new preset dictionaries; do not add scene-data mutations.
- To add new numeric channels, extend scene contract in `core/blender_gen.py` and bind in `render_batch.py` with explicit naming.
- Do not rename canonical objects/materials without updating:
  - `render_batch.py` contract validator
  - `validate_template_contract.py`
  - skill contract reference docs
