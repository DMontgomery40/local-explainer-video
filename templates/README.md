# Template Library

This directory stores deterministic slide assets used by the runtime template pipeline.

## Contract
- Backgrounds are static PNG assets at `1664x928`.
- Every background has a matching anchor file in `templates/anchors/`.
- `templates/manifest.json` is the machine-readable index used by the heuristic selector.
- Manifest entries must include release metadata:
  - `origin`: `qwen_curated` or `scaffold_only`
  - `operational_status`: `production` or `dev_only`
  - `production_ready`: explicit release gate
  - `selector_metadata`: human-readable predicate notes

## Naming Convention
- `template_id`: `<scene_type>_<layout_or_variant>_v<major>`
- Background file: `templates/backgrounds/<template_id>.png`
- Anchor file: `templates/anchors/<template_id>.json`

Examples:
- `bar_volume_chart_3panel_v1`
- `split_opposing_trends_v1`
- `title_card_v1`

## ID Stability
- Template IDs are immutable once referenced in committed plans.
- To replace visuals without breaking references, bump version (`..._v2`) and add a new manifest entry.
- Do not rename existing `template_id` values in place.

## Bootstrap Assets
- Starter assets are generated by `scripts/bootstrap_template_assets.py`.
- These are scaffolding defaults; curated production templates can replace them progressively.
- `generic_data_panel_v1` is development-only scaffolding and must not be used in production mode.
